## 1.3. 2023/11/13 nginx無法執行錯誤解決方案

> Q1：有同學建立第三台虛擬機發生以下錯誤，如下圖，可參考以下步驟解決


STEP1：在建立映像時，"自動在建立映像後刪除此虛擬機器"選項最好打勾，不然在啟動時有可能會有問題
[![image-20231113140020832](https://zanehsu.myqnapcloud.com:6876/uploads/images/gallery/2025-08/scaled-1680-/X9d686adc6d-image-20231113140020832.png)](https://zanehsu.myqnapcloud.com:6876/uploads/images/gallery/2025-08/X9d686adc6d-image-20231113140020832.png)


STEP2：進入lb後端集區，查看兩台VM是否都被加入後端集區，如果沒有請手動加入
[![image-20231113135215068](https://zanehsu.myqnapcloud.com:6876/uploads/images/gallery/2025-08/scaled-1680-/BAta3f23dd3-image-20231113135215068.png)](https://zanehsu.myqnapcloud.com:6876/uploads/images/gallery/2025-08/BAta3f23dd3-image-20231113135215068.png)


STEP3：請檢查下方操作步驟是否還有其他遺漏的部份

---

> Q2：原本設定的NAT規則明明沒更改，但是在建立第二台虛擬機後卻無法連線了


[![image-20231113135359803](https://zanehsu.myqnapcloud.com:6876/uploads/images/gallery/2025-08/scaled-1680-/nPI2f7e3a36-image-20231113135359803.png)](https://zanehsu.myqnapcloud.com:6876/uploads/images/gallery/2025-08/nPI2f7e3a36-image-20231113135359803.png)


STEP1：進入NAT輸入規則，將目標更改為VM2，網路IP位址也請除新選擇
[![image-20231113135055046](https://zanehsu.myqnapcloud.com:6876/uploads/images/gallery/2025-08/scaled-1680-/zA8b55a11cd-image-20231113135055046.png)](https://zanehsu.myqnapcloud.com:6876/uploads/images/gallery/2025-08/zA8b55a11cd-image-20231113135055046.png)


STEP2：更改完後，請輸入以下指令進行測試

```
ssh [你的公用IP] -l [你設定的帳號] -p [你設定的port]
```
[![image-20231113135523787](https://zanehsu.myqnapcloud.com:6876/uploads/images/gallery/2025-08/scaled-1680-/0KA2ea15172-image-20231113135523787.png)](https://zanehsu.myqnapcloud.com:6876/uploads/images/gallery/2025-08/0KA2ea15172-image-20231113135523787.png)


STEP3：登入成功後，可以用以下指令驗證nginx伺服器狀態
```
systemctl status nginx
```
[![image-20231113135643440](https://zanehsu.myqnapcloud.com:6876/uploads/images/gallery/2025-08/scaled-1680-/Kgn38459fa8-image-20231113135643440.png)](https://zanehsu.myqnapcloud.com:6876/uploads/images/gallery/2025-08/Kgn38459fa8-image-20231113135643440.png)
